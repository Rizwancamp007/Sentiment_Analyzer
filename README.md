# 🐦 Twitter Sentiment Analysis

> A complete end-to-end Machine Learning pipeline that classifies tweets as **Positive**, **Negative**, or **Neutral** — deployed as an interactive Flask web application.

---

## 📌 Overview

This project builds a sentiment analysis system from scratch using a real-world Twitter dataset from Kaggle (~27,000 labelled tweets). It covers the full ML pipeline — data exploration, text preprocessing, model training, evaluation, and deployment via a Flask web app where users can type any sentence and get an instant sentiment prediction.

---

## 📂 Dataset

| Property | Details |
|---|---|
| Total Rows | 27,481 tweets |
| Total Columns | 10 |
| Sentiment Classes | Positive, Negative, Neutral |
| Positive Tweets | 8,582 (31.2%) |
| Negative Tweets | 7,781 (28.3%) |
| Neutral Tweets | 11,118 (40.5%) |
| Missing Values | 1 row (dropped) |
| File Encoding | `latin-1` |
| File Size | ~4.6 MB |

Only two columns are used for modelling: `text` (tweet content) and `sentiment` (label).

## 🔬 Notebook Walkthrough

### Cell 1 — Libraries & Setup
Imports all dependencies: `pandas`, `numpy`, `matplotlib`, `seaborn`, `nltk`, `scikit-learn`, and `joblib`. Downloads NLTK's stopword list for text cleaning.

### Cell 2 — Loading the Dataset
Loads the CSV with `latin-1` encoding to handle special characters in tweets, then previews shape and structure.

### Cell 3–5 — Exploratory Data Analysis (EDA)
- Sentiment distribution (bar chart)
- Null value check (1 row dropped)
- Data type validation

> **Key finding:** Dataset is slightly imbalanced — neutral tweets lead at 40.5% — but not enough to cause significant issues.

### Cell 6 — Word Clouds
Generates one word cloud per sentiment class to visually confirm patterns:
- **Positive** → *love, amazing, great, fantastic*
- **Negative** → *worst, terrible, awful, disappointed*
- **Neutral** → *available, includes, hours, located*

### Cell 7–8 — Text Preprocessing
A `clean_text()` function handles all noise removal:
- Lowercasing
- Removing URLs, @mentions, #hashtags
- Stripping punctuation and numbers
- Removing stopwords
- Dropping tokens shorter than 3 characters

> Interesting insight: negative tweets tend to be slightly longer — people write more when they're unhappy!

### Cell 9 — Train/Test Split & TF-IDF Vectorization
Converts tweets to numeric vectors using **TF-IDF** (Term Frequency–Inverse Document Frequency):
- **TF** — how often a word appears in a tweet
- **IDF** — penalises words common across all tweets
- Words unique to a tweet get higher scores

Data split: **80% train / 20% test** with `stratify=y` to preserve class balance.

### Cell 10 — Training Three Models

| Model | Why We Use It | Strength |
|---|---|---|
| Logistic Regression | Simple, fast baseline for text | Interpretable & reliable |
| Naive Bayes | Classic NLP algorithm | Fast training, handles small data well |
| Linear SVM | Top performer for text classification | Finds optimal decision boundary |

### Cell 11–13 — Evaluation

| Metric | Description |
|---|---|
| **Accuracy** | Overall % of correct predictions |
| **Precision** | Of predicted positives, how many were actually positive? |
| **Recall** | Of all actual positives, how many did we find? |
| **F1-Score** | Harmonic mean of Precision and Recall |

### Cell 14–15 — Saving the Model
Saves two files using `joblib`:
- `sentiment_model.pkl` — trained classifier
- `tfidf_vectorizer.pkl` — fitted TF-IDF transformer

Followed by a quick sanity check on 3 new sentences.

## 🌐 Flask Web App

The app exposes two routes:

| Route | Method | Description |
|---|---|---|
| `/` | GET | Serves the main web page |
| `/predict` | POST | Accepts JSON `{ text }`, returns sentiment + confidence scores |

**Prediction flow:**
1. User types text → clicks *Analyze Sentiment*
2. JavaScript sends a POST request to `/predict`
3. Flask cleans the text using `clean_text()`
4. Cleaned text is TF-IDF vectorized
5. Model predicts sentiment + confidence scores
6. Result is returned as JSON and rendered with animated confidence bars

## 📊 Model Performance

| Model | Accuracy | Notes |
|---|---|---|
| Logistic Regression | ~82–85% | Strong baseline, fast to train |
| Naive Bayes | ~78–82% | Slightly lower but very fast |
| Linear SVM | ~83–86% | Usually the best performer for text |

The best model is automatically saved as `sentiment_model.pkl`. Flask always uses this model — no manual selection needed.

## 🚀 Getting Started

### Step 1 — Install Dependencies
```bash
pip install pandas scikit-learn nltk matplotlib seaborn wordcloud flask joblib python-docx
```

### Step 2 — Run the Jupyter Notebook
Open `sentiment_analysis.ipynb` and run all cells top to bottom. This will:
- Train all three models
- Save `sentiment_model.pkl` and `tfidf_vectorizer.pkl`
- Generate EDA charts

### Step 3 — Launch the Web App
```bash
python app.py
```
Then open your browser at: **http://127.0.0.1:5000**

### Step 4 — Use the App
- Type or paste any text into the input box
- Click **Analyze Sentiment** or press `Ctrl+Enter`
- View the predicted sentiment and confidence scores
- Try the quick-example chips for instant demos
